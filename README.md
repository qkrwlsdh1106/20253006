# Linux 명령어 top, ps, jobs, kill

# 1. top
   * **'top'** 명령어는 리눅스 시스템에서 실행 중인 프로세스와 시스템의 자원 사용 현황을 실시간으로 보여줍니다.

### 기본 사용법
  * 터미널에서 **'top'** 명령어 입력하고 엔터를 누르면 됩니다.

### 주요 기능
  * **실시간 모니터링**: 'top' 명령어는 일정 주기로 화면을 업데이트하여 시스템 성능을 실시간으로 모니터링 할 수 있습니다.
  * **프로세스 정보**: 실행 중인 프로세스의 PID, 사용자, 우선순위, 메모리 사용량, CPU사용량 등을 보여줍니다.
  * **시스템 요약**: 화면 상단에 시스템의 전반적인 성능 지표를 요약하여 표시합니다. 이는 시스템 가동 시간, 사용자 수, 로드 평균, CPU 상태, 메모리 사용량, 스왑 공간 사용량 등을 포함합니다.

### 주요 옵션
  |옵션|설명|
  |----|----|
  |-d delay|지정한 시간 간격으로 정보를 출력|
  |-n num|지정한 횟수만큼 정보를 출력|
  |-p pid|지정한 프로세스 id의 정보만 출력|

# 2. ps
  * **'ps'** 명령어는 Process State의 약자로 현재 실행중인 프로세스와 상태를 출력하는 명령어 입니다.

### 기본 사용법
  * 터미널에서 **'ps'** 명령어를 입력하고 엔터를 누르면 됩니다.

### 주요 옵션
  |옵션|설명|
  |----|----|
  |-p|프로세서 아이디|
  |-e|현재 실행 중인 모든 프로세서|
  |-f|실제 유저명, 개시 시간 등을 표시|
  |-l|프로세서의 상태, 우선도 등과 같은 상세한 정보를 표시|
  |-A|모든 프로세스를 출력|
  |-u username|특정 사용자가 실행한 프로세스 표시|
  |-p pid|지정한 프로세스 id의 정보만 출력|
  |-ef --forest|프로세스 간의 부모-자식 관계를 트리 구조로 표시|

# 3. jobs
  * **'jobs'** 명령어는 현재 쉘 세션에서 실행 중인 백그라운드 및 포그라운드 작업의 목록을 표시하는 명령어 입니다. 이 명령어는 주로 작업 제어와 관련된 정보를 제공하며 사용자에게 어떤 작업이 실행 중인지, 멈춰 있는지 등을 확인할 수 있게 해줍니다.

### 기본 사용법
  * 터미널에서 **'jobs'** 명령어를 입력하고 엔터를 누르면 됩니다.

### 주요 옵션
  |옵션|설명|
  |----|----|
  |-l|PID 포함 상세 정보|
  |-r|Running 상태만 표시|
  |-s|Stopped 상태만 표시|

### 사용화면 및 요소 설명
  <img width="382" height="77" alt="스크린샷 2025-11-28 191443" src="https://github.com/user-attachments/assets/27b1ec47-5b9f-486f-98e7-8a596bde8e57" />
  
  * **[1]**
    * 쉘 내부에서 관리하는 작업 번호

  * **+/-**
    * **+**: 기본 선택 작업 (우선)
    * **-**: 그 다음 순위

  * **Running/Stopped/Done/Done(code)/Terminated**
    * **Running**: 작업이 계속 진행중임
    * **Stopped**: 작업이 일시 중단
    * **Done**: 작업이 완료되어 0을 반환
    * **Done(code)**: 작업이 종료되었으며 0이 아닌 코드를 반환
    * **Terminated**: 작업이 강제 종료됨

  * **sleep 100**
    * 실제 실행 중인 명령 이름

  * **&**
    * 백그라운드 실행을 의미

# 4. kill
  * **'kill'** 명령어는 주로 프로세스가 더 이상 필요하지 않거나 비정상적으로 동작할 때 사용됩니다.

### 기본 사용법
  * **'kill'** 명령어는 kill [option]<pid> 방식으로 사용됩니다.

### 주요 옵션
  |옵션|설명|
  |----|----|
  |-s|특정 시그널을 사용하여 프로세스를 종료, 기본적으로 SIGTERM 시그널이 적용|
  |-l, --list|지원되는 시그널 목록을 출력|
  |-a, --all|현재 사용자에 속한 모든 프로세스를 종료|
  |-q, --queue|프로세스에 시그널을 보내는 대신 시그널을 대기열에 추가|
